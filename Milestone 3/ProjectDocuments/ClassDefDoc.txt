CLASS: UVSim
----------------------------------------------------------------
PURPOSE: 
    Acts as the central processing unit and memory storage for the UVSim virtual machine. 
    It manages the 100-word memory array, the accumulator register, and the execution 
    logic for all Basic Machine Language (BML) opcodes (I/O, Load/Store, Arithmetic, 
    and Control Transfer).

FIELDS:
    1. memory (string array[100]): Stores instructions and data as 5-character signed strings.
    2. accumulator (int): A central register used for arithmetic operations and data loading.
    3. address (int): Acts as the instruction pointer or program counter.
    4. halted (bool): Tracks the execution state of the simulator.

FUNCTIONS:
    1. get
         Purpose: Retrieves a word from the specified memory address with bounds checking.
         Parameters: 
             memory (string array): The current memory state.
             memory_address (int): The index to access.
         Return Value: string (the word at that address).
         Pre-condition: memory_address should be within [0, 99].
         Post-condition: Returns the string value or terminates program on out-of-bounds access.

    2. READ
         Purpose: Prompts user for a signed 4-digit integer to store in memory.
         Parameters: 
             memory_address (int): The location where the input will be stored.
         Return Value: None.
         Pre-condition: memory_address must be valid.
         Post-condition: Memory at the specified address is updated with a validated 5-character string.

    3. WRITE
         Purpose: Outputs the word stored at a specific memory address to the console.
         Parameters: 
             memory_address (int): The location to read from.
         Return Value: None.
         Pre-condition: memory_address must be valid.
         Post-condition: The value is displayed to the user via standard output.

    4. LOAD
         Purpose: Moves a value from memory into the accumulator.
         Parameters: 
             address (int&): Reference to the current instruction address to find the operand.
             accumulator (int&): Reference to the accumulator to be updated.
         Return Value: None.
         Pre-condition: The operand address defined in the instruction must contain a valid number.
         Post-condition: Accumulator is updated with the integer value from memory.

    5. STORE
         Purpose: Moves the value in the accumulator into a specific memory location.
         Parameters: 
             address (int&): Reference to the current instruction address to find the operand.
             accumulator (int&): Reference to the value to be stored.
         Return Value: None.
         Pre-condition: Accumulator holds a valid integer.
         Post-condition: Memory at the operand address is updated with a formatted 5-character string.

    6. ADD / SUBTRACT / MULTIPLY / DIVIDE
         Purpose: Performs the specified arithmetic operation between the accumulator and a memory value.
         Parameters: 
             accum (int): The current value in the accumulator.
             location (string): The memory address (as a string) containing the second operand.
         Return Value: int (the result of the calculation).
         Pre-condition: Memory location must contain numeric data; DIVIDE requires a non-zero divisor.
         Post-condition: Returns the result to be placed back into the accumulator; throws runtime_error on invalid math.

    7. BRANCH / BRANCHNEG / BRANCHZERO
         Purpose: Handles control flow by updating the instruction address based on conditions.
         Parameters: 
             accumulator (int): Value to check for conditions (NEG/ZERO).
             old_address (int): The current address to return if the condition is not met.
             memory_address (int): The target address to jump to.
         Return Value: int (the new address the program should move to).
         Pre-condition: memory_address must be a valid index.
         Post-condition: Returns either the target branch address or the current address to continue sequence.

CLASS: Console
----------------------------------------------------------------
PURPOSE: 
    Provides a modular logging system to decouple the user interface's output display 
    from the core simulation logic.

FIELDS:
    1. logs (vector<string>): A collection of historical output messages.

FUNCTIONS:
    1. addLog
         Purpose: Appends a new message to the log history.
         Parameters: 
             message (const string&): The text to be stored.
         Return Value: None.
         Pre-condition: None.
         Post-condition: The logs vector size increases by one.

    2. clear
         Purpose: Erases all stored logs.
         Parameters: None.
         Return Value: None.
         Pre-condition: None.
         Post-condition: The logs vector is empty.

    3. getLogs
         Purpose: Provides read-only access to the log collection.
         Parameters: None.
         Return Value: const vector<string>& (reference to the list of strings).
         Pre-condition: None.
         Post-condition: Returns the current state of logs without allowing modification.

CLASS: Window
----------------------------------------------------------------
PURPOSE: 
    Allows the GUI to have multiple Windows. Keeps track of things like which UVSim it has,
    if the background is green, whether it is stopped or running, and other things.

FIELDS:
    1. Stopped(bool): To see if the window is stopped.
    2. Running(bool): To see if the window is running.
    3. timer(float): To keep time for running the program.
    4. windowSim(UVSim): The simulator to deal with the users input.
    5. readInput(string): To take in the user's input.
    6. waitingForRead(bool): A check to see if it is waiting for the user's input or not.
    7. openFile(bool): To see if a txt file has been opened.
    8. done(bool): To see if program is done.
    9. consoleInput(string): For the console input. 
    10. isBackgroundGreen(bool): To see if background should be green or not.
FUNCTIONS:
    1. incrementTime
         Purpose: Increments the time on the timer.
         Paramaters: 
            time(float): an amount of time for the class to increment the timer.
         Return Value: None.
         Pre-condition: Timer must be initialized. 
         Post-condition: The new, updated time on the timer.
    2. execute
         Purpose: Executes the UVSim's method, Execute, which carries out the logic of UVSim.
         Paramaters: None.
         Return Value: bool
         Pre-condition: The window must exist.
         Post-condition: Changes the UVSim to execute whatever program is in the memory.
    3. isRunning
         Purpose: Checks if the program is running, stopped, 
         and waiting for read. If it is running and not stopped and not waiting for read, then it returns
         true.
         Paramaters: None.
         Return Value: bool
         Pre-condition: The window must exist.
         Post-condition: None.
    4. getRunning
         Purpose: The getter to see if the window is running. Returns true when the window is running.
         Paramaters: None.
         Return Value: bool. True if the window is running, false if it is not.
         Pre-condition: The window must exist.
         Post-condition: None.
    5. setRunning
         Purpose: Deals with setting if running is true or false.
         Paramaters: 
            myBool(bool): The new value to set Running to.
         Return Value: None.
         Pre-condition: The window must exist.
         Post-condition:
    6. getStopped
         Purpose: To check if the window is currently stopped.
         Parameters: None.
         Return Value: bool.
         Pre-condition: The window must exist.
         Post-condition: Nothing changes; it just reports the current status.
    7. setStopped
         Purpose: To update the stopped status of the window.
         Parameters: myBool(bool)
         Return Value: None.
         Pre-condition: The window must exist.
         Post-condition: The window's Stopped status is now set to whatever value was passed in (myBool).
    8. getTimer
         Purpose: To get the current time value stored in the window.
         Parameters: None.
         Return Value: A float representing the time.
         Pre-condition: The window and its timer variable must be initialized.
         Post-condition: The window stays the same; it just returns the current timer value.
    9. setTimer
         Purpose: To set or update the window's timer to a specific value.
         Parameters: newTime(float).
         Return Value: None.
         Pre-condition: The window must exist.
         Post-condition: The timer variable is updated to the value of newTime.

CLASS: Tab
----------------------------------------------------------------
PURPOSE: 
    Keeps track of a Window and keeps track of the tabs ID.

FIELDS:
    1. open (bool): A flag indicating if the tab is currently active in the UI. When false, the tab is flagged for removal from the session.
    2. tabWindow (Window): The core simulation instance and GUI state associated with this specific tab.
    3. nextTabId (inline static int): A shared counter used to assign unique identifiers to every new tab instance.
    4. id (int): The unique numerical identifier for the specific tab instance.

FUNCTIONS:
    1. Tab (Constructor)
        Purpose: Initializes a new tab instance and ensures it receives a unique ID.
        Parameters: None.
        Return Value: N/A (Constructor).
        Pre-condition: None.
        Post-condition: The id is set to the current value of nextTabId, and nextTabId is incremented by one.
